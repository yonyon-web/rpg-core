# アーカイブドキュメント

このディレクトリには、過去の分析文書、計画書、評価文書などが保存されています。
これらの文書は歴史的な参考資料として残されており、現在のプロジェクトには直接影響しませんが、
設計判断の背景や検討プロセスを理解する際に役立ちます。

## 📋 含まれる文書

### 依存性注入分析（Dependency Injection Analysis）

- **DEPENDENCY_INJECTION_ANALYSIS.md** - サービス関数の依存関係パラメータ分析
- **DEPENDENCY_INJECTION_ANALYSIS_RESULT.md** - 分析結果の詳細
- **DEPENDENCY_INJECTION_SUMMARY.md** - 分析のサマリーと結論

**背景**: DIコンテナ導入時に、どのパラメータを自動注入すべきか、どれを明示的に渡すべきかを分析した文書。現在の設計パターンの根拠となっています。

### srcディレクトリ再構成提案（SRC Reorganization）

- **SRC_REORGANIZATION_INDEX.md** - 再構成提案のインデックス
- **SRC_REORGANIZATION_PROPOSALS.md** - 4つの提案の詳細
- **SRC_REORGANIZATION_SUMMARY.md** - 提案の要約と比較
- **SRC_REORGANIZATION_VISUAL.md** - ビジュアル図解

**背景**: プロジェクトの成長に伴い、srcディレクトリの構造を見直すための複数の提案を作成。それぞれのメリット・デメリットを比較検討した文書。

### 実装計画書（Implementation Plans）

- **IMPLEMENTATION_PLAN_SUMMARY.md** - ヘッドレスUI実装計画の作成完了報告

**背景**: ヘッドレスUI実装計画書の作成時に、作成内容をサマリーとして記録した文書。

### 評価文書（Evaluations）

- **DEPENDENCY_MANAGEMENT_EVALUATION.md** - 依存関係管理アプローチの評価

**背景**: RPGCoreクラスとServiceContainerの導入時に、アプローチの妥当性を評価した文書。

## 🔍 参照方法

これらの文書は、以下のような場合に参照すると有益です：

1. **設計判断の背景を知りたいとき**
   - なぜ現在の設計になったのか
   - 他にどんな選択肢があったのか

2. **同様の課題に直面したとき**
   - 過去の検討プロセスを参考にできる
   - 同じ議論を繰り返さなくて済む

3. **プロジェクトの歴史を理解したいとき**
   - どのような進化を遂げてきたか
   - 重要な意思決定のタイミング

## ⚠️ 注意事項

- これらの文書は**参考資料**であり、現在の実装とは異なる可能性があります
- 最新の情報は、メインのドキュメントとソースコードを参照してください
- 新しい設計を検討する際の参考として活用してください
