## 🎯 活用方法

### 開発者向け

これらの文書は以下のような場合に参照してください：

1. **現在の実装状況を確認したいとき**
   - どの機能が実装済みか
   - どの機能がまだ未実装か
   - 各機能の完成度

2. **次に取り組むべき作業を決めるとき**
   - 優先度の高い課題
   - 技術的負債の状況
   - ドキュメントの更新が必要な箇所

3. **プロジェクトの進捗を報告するとき**
   - 完了したマイルストーン
   - 現在進行中の作業
   - 今後の予定

### メンテナー向け

- 実装状況を定期的に更新してください
- 新しい課題が見つかったら「次に考えるべき課題.md」に追加
- 完了した課題はチェックマークをつけて、完了日を記録

## 📝 更新ガイドライン

### 実装状況.mdの更新

新しい機能を実装したら：

1. 該当する機能のステータスを更新
   - 🔴 未着手 → 🟡 進行中 → 🟢 完了 → ✅ テスト済み
2. 実装ファイルのパスを記載
3. テストの有無を更新
4. 「最終更新」日付を更新

### 次に考えるべき課題.mdの更新

新しい課題を追加するとき：

1. 適切なカテゴリに追加（ドキュメント、技術、機能など）
2. 優先度を設定（High/Medium/Low）
3. 具体的なタスクリストを作成
4. 関連するIssue番号があれば記載

課題を完了したとき：

1. チェックボックスをチェック
2. ✅ 完了マークを追加
3. 完了日や簡単な完了報告を記載
